# Contributing to WDAC Repository

Thank you for your interest in contributing to the WDAC Repository! This document provides guidelines and best practices for contributing to this project.

## How to Contribute

### Reporting Issues

If you find a bug, have a question, or want to suggest an enhancement:

1. Check the existing issues to avoid duplicates
2. Create a new issue with a descriptive title
3. Provide as much relevant information as possible
4. Include steps to reproduce the issue if applicable

### Submitting Changes

1. Fork the repository
2. Create a new branch for your changes
3. Make your changes following our style guides
4. Ensure all tests pass
5. Commit your changes with a clear, descriptive message
6. Push your changes to your fork
7. Submit a pull request to the main repository

## Style Guides

### Documentation Style

- Use clear, concise language
- Follow Microsoft's style guide for technical documentation
- Use proper Markdown formatting
- Include code examples where appropriate
- Update table of contents when adding new documents

### PowerShell Script Style

- Follow Microsoft's PowerShell Best Practices and Style Guide
- Use meaningful variable and function names
- Include comment-based help for all functions
- Use appropriate error handling
- Test scripts in multiple Windows versions when possible

### XML Policy Style

- Follow WDAC XML schema standards
- Use meaningful IDs and names
- Include descriptive comments
- Maintain consistent formatting

## Code of Conduct

This project follows a code of conduct adapted from the Contributor Covenant. By participating, you are expected to uphold this code.

## Questions?

If you have any questions about contributing, please open an issue or contact the maintainers.

Thank you for helping improve the WDAC Repository!
